What does Quality Assurance mean for web site development?
Quality Assurance (QA) guarantees that users get the best user experience and a stable user interface when using a mobile app or website. It is like a vaccine to prevent users from encountering inconveniences or flaws that might have been missed in the process of designing and developing by testing the site in various ways, whether the site is working well on any device or not. This usually involves testing requirements, evaluating designs, and testing functions.

Requirement testing takes into account the customer's requirements as well as other design documents approved by the customer. In this process, it should be evaluated that the layout of the site or the prototype version of the site meets the customer's requirements.

Design testing refers to looking at the appearance of the website. The design of the web app or website is checked whether their look and feel match the proposed design and the overall layout or not.

Functional testing is a process of ensuring that all of the functions on the website are properly executed as expected. It is the process of ensuring that the buttons or links on a website work well and that they operate properly without bugs in other browsers or devices.
Describe the two key principles of Quality Assurance: "Fit for Purpose" and "Right First Time."
“Fit for purpose” means that the product or service meets its intended purpose. And “right first time” is that any defect should be dealt with immediately.

With the two above principles in play, the goal is to ensure that the product or service works correctly all the time by controlling the project variables. To do so, quality assurance involves the management of the quality of raw materials, assemblies, products and components; services related to production; and the management, production and inspection process.
How do web standard relate to web quality?
Web standards are rules and guidelines established by the World Wide Web Consortium (W3C) developed to promote consistency in the design code which makes up a web page. It simply refers to the mark-up language rule that defines how a web page is presented in the browser window of a visitor. There are several advantages to complying with these standards.

First, web pages will display in a wide variety of browsers and computers, including new technology like iPhones, Droids, iPads, PDA devices, mobile phones, which greatly increases the viewing audience. Second, W3C Standards promote the use of "Cascading Style Sheets" (CSS) or design code which is attached to the web page. rather than embedded in the page. The use of style sheets significantly reduces the page file size which means not only a faster page loading time but lower hosting costs for frequently visited sites due to using less bandwidth. Third, Design features such as colors and fonts can be easily changed by just modifying one style sheet instead of editing every individual page in a web site, reducing the costs to modify your site. Fourth, Search Engines are able to access and index pages designed to web standards with greater efficiency. Therefore, keeping standard will ultimately improve the quality of the website.
Describe what Web Accessibility means. What steps you can take to make your pages more accessible?
Web accessibility means that websites, tools, and technologies are designed and developed so that people with disabilities can use them. It ensures that there are no barriers that prevent interaction with, or access to, websites on the World Wide Web by people with physical disabilities, situational disabilities, and socio-economic restrictions on bandwidth and speed. When sites are correctly designed, developed and edited, generally all users have equal access to information and functionality. To achieve web accessibility, you can take the following steps:
1. Understand what is accessibility
Start by recognizing what accessibility is. Most people don't make it because they lack awareness, not because they oppose it. It is important to understand what accessibility is and what factors we should care about in order to make our website accessible.
2. Build a document structure appropriately
Headings, lists, and other structural elements provide meaning and structure to web pages. They can also facilitate keyboard navigation within the page.
3. Use alternative text when inserting an image
Alternative text provides a textual alternative to non-text content in web pages. It is especially helpful for people who are blind and rely on a screen reader to have the content of the website to read to them. When creating the alt text, the text should contain the message you want to convey through that image, and if the image includes text, that text should also be included in the alt.
4. Use the right word in the link
Every link should make sense if the link text is read by itself. Screen reader users may choose to read only the links on a web page. Certain phrases like "click here" and "more" must be avoided. Rather, it is better to use specific and distinct words.
5. Use the color carefully
People with disabilities who do not recognize a particular color or those with learning disabilities are affected by specific colors. Color must be used carefully to satisfy these people. A tool to evaluate colors will help you know if you have used colors correctly.
6. Use the table for data
Tables should be used to organize the data to suit its original purpose. In order to create a layout of website, use CSS instead of a table. The use of too many tables will bother people using screen readers. In addition, the table headers should be properly used in the table, and the headers should be able to describe their relationship with the cell.
7. A website should be able to navigate with the keyboard
A website should be made so that it can be explored only with a keyboard, taking into account those who are impaired in mobility or have been injured. Pages should be possible to distinguish sections by anchor links or to skip specific sections. Don't use hover css on the important elements. If you make important content visible only when a user hovers over it with a mouse, people using only the keyboard or users using a screen reader can't see the content.
